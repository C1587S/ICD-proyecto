La base de datos que se utilizar contiene informacin sobre los procedimientos de interrupcin legal del embarazo (ILE) que se realizaron en las clnicas y hospitales de la Secretara de la Salud de la Ciudad d Mxico a partir de 2016. Los datos son pblicos y se encuentran en la pgina web de Datos Abiertos Ciudad de Mxico.

La base de datos analizada es de frecuencia mensual, iniciando en abril de 2016 y finaliza en junio de 2019, e incluye las siguientes variables:

  • year : Ao en que se realiz el procedimiento
  • mes : Mes en que se realiz el procedimiento
  • _cvehospital : Clave de la Unidad Mdica donde se realiz el procedimiento
  • fingreso : Fecha de Interrupcin Legal del Embarazo
  • autoref : Especificar si la paciente es referida de otra unidad o acude directamente
  • _edocivildescripcion : Estado civil
  • edad : Edad cumplida en aos
  • _descderechohab : Especificar la institucin que otorga la derechohabiencia
  • _niveledu : ltimo nivel escola acreditado
  • ocupacion : Ocupacin
  • religion : Religin
  • parentezco : Parentesco del responsable con la paciente
  • entidad : Entidad de residencia
  • _alc_omunicipio : Delegacin o municipio de residencia
  • menarca : Edad de la primera menstruacin
  • fsexual : Edad de inicio de vida sexual activa
  • fmenstrua : Fecha de la ltima menstruacin
  • sememb : Semanas de embarazo por fecha de ltima menstruacin
  • nhijos : Nmero de hijos
  • gesta : Nmero de embarazos (Incluyendo abortos)
  • naborto : Nmero de abortos
  • npartos : Nmero de nacimientos por va vaginal
  • ncesarea : Nmero de cesreas
  • nile : Nmero de Interrupciones Legales del Embarazo previas
  • consejeria : Especificar si la paciente recibi consejera sobre la ILE
  • anticonceptivo : Especificar si la paciente utiliza de forma habitual mtodo anticonceptivo
  • _cnum : Nmero de consultas
  • motiles : Motivo por el cual se desea la interrupcin del embarazo
  • _hingreso : En caso de hospitalizacin, fecha de ingreso
  • _descservicio : Servicio en el que se otorg la interrupcin
  • _psemgest : Semanas de gestacin por USG
  • _pdiasgesta : Das de gestacin por USG
  • _pconsent : Se requisit y firmo el consentimiento informado
  • procile : Mtodo utilizado para la interrupcin legal
  • _scomplica : Se presentaron complicaciones por el procedimiento
  • panticoncep : Mtodo anticonceptivo proporcionado post evento obsttrico

La variable de respuesta que se quiere predecir es una variable binaria que indica si la edad es o no mayor de 23 aos ($\geq23$aos). En particular, se quiere predecir si dadas ciertas caractersticas de la persona que llega a realizarse un aborto su edad es mayor o igual a 23 aos, o no.

Nota: la variable de respuesta se transforma antes de partir el set de datos?

En el presente ejercicio de anlisis estadstico se realiza el siguiente procedimiento:

  • Inicialmente, se realiza la limpieza, y organizacin de la base de datos a utilizar.
  • Se realiza un ejercicio de data profiling,
  • Se hace un ejercicio explorationio de los datos (GEDA) destacando las 5 caractersticas ms importantes del conjunto de datos.
  • Se realiza feature engineering para seleccionar las caractersticas ms importantes para explciar la variable explicada.
  • Finalmente, se genera un magic loop incluyendo un hiperparameter tunninh con 3 modelos: rbol, RandomForest, y XGBoost (GradientBoostingClassifier).

Implicaciones éticas¶

Fase Implicacin
Fase I Data profiling y seleccin de variables por relevacia (datos faltantes). En esta fase del proceso se realiz el profiling de la base de datos y se determinaron ciertos criterios para utilizar o no ciertas variables de manera inicial. Inicialmente, se determin que no se utilizaran los datos que tuviesen menos mns del 10% de datos como datos faltantes. Este crterio de decisin puede traer implicaciones ticas cuando se quitan caractersticas que podran ser tlies para la clasificacin en el problema, o que podran determinar que la llegada o no de los anticonceptivos de un posible programa de salud llegasen a la poblacin. Sin embargo, tambin se realiz este tipo de procedimiento para poder procesar mejor los datos y disponibles y as realizar una mejor prediccin futura.,Es especial queremos mencionar dos variables que pueden tener implicaciones concretas:,c_num - el nmero de consultas pudiera ser un indicador de algn problema con algna mujer, al quitar esta variable removemos esa posibilidad. La variable ocupacin podra generar implicaciones ticas al no tomar en cuenta grupos que son ms vulnerables.Sin embrago la implicacin tica profunda se hace evidente cuando se determina el objetivo de la clasificacin y notamos los casos que quedaron fuera debido a problemas en el conjunto de datos.Las variables eliminadas son presentadas en el desarrollo del documento.
Fase I Data profiling y seleccin de variables por relevacia (alta cardinalidad) De manera similar al enunciado anterior, tambin se utiliz como criterio de seleccin de variables un nivel de cardinalidad menor a 20, lo anterior, en la medida de evitar problemas de procesamiento de los datos que permitan identificar las mejores caractersticas para predecir la variable de repuesta. Por ejemplo, alcalda, municipio y religin fueron eliminadas. Sin embargo, es muy claro que este tipo de variables podran explicar algunas caractersticas que llevasen a una mujer mayor a 23 aos a realizar una intervencin legal del embarazo.En este sentido, al igual que en el caso anterior, eliminar estas variables en el anlisis puede implicar crear algn sesgo en la entrega, por ejemplo de anticonceptivos en la poblacin, lo que podra afectar las vidas de algunas personas, y sus gastos.Las variables eliminadas son presentadas en el desarrollo del documento.
Fase II En similitud con la fase I del proceso en este paso tambin se eliminan varias caractersticas de la poblacin.,En particular, queremos resaltar variables categricas como el estado civil y Servicio en el que se otorg la interrupcin. En particular, dados los criterior utilizados estas features no resultadon relevantes. Sin embargo, por ejemplo, si una persona est casada posiblemente tenga el apoyo de su pareja para adquirir mtodos anticonceptivos, mientras que una soltera no. En dicho sentido, eliminar esta variable para predecir menores de 23 aos que podran recibir por ejemplo apoyo para adquirir anticonceptivos, podra omitir estas implicaciones econmicas.
Fase III La fase ms relevante es aquella en la cual ya hemos seleccionado el modelo para realizar prediccin, y ms importante an, qu decisiones vamos a tomar con las predicciones realizadas. Considerando los ejemplos que se mencionan en el desarrollo del anlisis, si una farmacutica o un gobierno utiliza el modelo para dar preservativos o generar programas de salud a las mujeres que el modelo clasifique como menores de 23 con el fin de evitar que posteriormente incurran en una ILE,si se realizan malas predicciones, se podra afectar la probabilidad que algunas mujeres en un futuro puedan incurrir en dichos procedimientos.
